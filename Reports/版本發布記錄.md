## 🔐 安全檢查規範

### Commit Message 安全準則

**❌ 禁止包含**:
- 明確的檔案名稱列表
- 詳細的目錄結構
- 內部模組路徑
- 具體的類別/函數名稱
- 技術實作細節

**✅ 應該包含**:
- 功能性描述
- 使用者可見的改進
- 版本號變更
- 重大功能更新概述

**良好範例**:
```
v1.0.6 - 多語言支援與性能優化

- 新增 4 種語言介面支援
- 優化批次處理與快取機制
- 改善 MCP 系統整合
- 更新文件與依賴項
```

**不良範例**:
```
❌ 新增 gemini_async_batch_processor.py 與 utils/memory_cache.py
❌ 修改 gemini_chat.py 的 line 125-134
❌ 整合 locales/zh_TW.yaml, en.yaml, ja.yaml, ko.yaml
```

### 推送前檢查清單

**每次推送前必須確認**:
- [ ] Commit message 無檔案名稱
- [ ] Commit message 無內部路徑
- [ ] 版本號已更新
- [ ] 開發文件已移除（Phase*.md、*SPEC.md、OPTIMIZATION*.md 等）
- [ ] 測試檔案已整理至 tests/
- [ ] README 依賴的文件已確認保留（最多 5 個）
- [ ] 無本地路徑洩露（檢查所有 .md 文件）
- [ ] .gitignore 包含所有敏感檔案

---

### 緊急撤銷推送流程

**情境**: 推送後發現包含隱私文件

**步驟**:
```bash
# 1. 移除隱私文件
mv Phase*.md OPTIMIZATION*.md ROLLBACK*.md ~/Saki_Studio/Claude/

# 2. 撤銷本地 commit
git reset --soft HEAD~1

# 3. 重新 commit（排除隱私文件）
git add -A
git commit -m "v1.0.x - 功能更新"

# 4. 強制推送覆蓋遠端
git push --force origin main
```

**⚠️ 重要提醒**:
- 使用 `--force` 會覆蓋遠端歷史，僅限緊急情況
- 確認無其他協作者正在使用該分支
- 推送前再次檢查隱私文件已移除
